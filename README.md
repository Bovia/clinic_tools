门诊患者“诊疗进度陪伴”小程序（MVP版本） 可能是一个 小程序

#### 1. 项目背景与目标

- **痛点：** 患者术后咨询多、复诊流失率高、人工提醒耗时，且传统话术难以提升门诊的专业感知度。
- **目标：** 开发一套基于微信的小程序，将“注意事项”变成“任务进度条”，通过“打卡”机制提升患者粘性，减轻前台人工回访压力。

#### 2. 核心用户流程

1. **开单录入（后台）：** 医生/前台在本地系统选择患者及治疗项目（如：种植牙），系统自动匹配对应的“诊疗任务包”。
2. **触发提醒（触达）：** 系统通过微信推送/小程序通知，发送对应的“诊疗进度卡片”。
3. **陪伴交互（用户端）：** 患者每日查看任务（如：今日冰敷），点击“已完成”，获得正向反馈（电商感的进度动画）。
4. **自动转化（闭环）：** 任务临近结束时，小程序自动推送“复诊预约”入口。

#### 3. 功能需求清单


| **功能模块**     | **需求描述**                   | **优先级** |
| ------------ | -------------------------- | ------- |
| **诊疗进度看板**   | 首页展示当前项目的进度条（如：已完成 2/5 步）  | P0      |
| **任务动态卡片**   | 每日推送当日注意事项，支持图片与简短说明       | P0      |
| **打卡互动**     | 完成任务后点击按钮，显示“已完成”视觉反馈      | P1      |
| **一键沟通**     | 悬浮“一键拨号”与“在线咨询”按钮          | P0      |
| **后台配置（精简）** | 预设不同治疗项目的任务时间轴（如拔牙第1-7天内容） | P0      |


#### 4. UI/交互风格（电商化）

- **视觉规范：** 采用简洁的圆角卡片式设计（类似淘宝/小红书风格）。
- **交互逻辑：** 底部悬浮固定“服务热线”按钮；进度条采用动态填充效果。
- **反馈机制：** 每次点击“已完成”，增加一个清脆的“叮”声或视觉点赞效果。

#### 5. 数据逻辑（本地轻量化）

- **存储：** 采用微信云开发（无需服务器运维，数据自动存储于腾讯云，安全性高）。
- **数据模型：**
  - `Project_Template`：配置不同病种（种植、拔牙、根管）的流程节点。
  - `Patient_Diary`：记录患者 ID、所属项目、当前进度节点、任务完成时间。

#### 6. 成功评估标准（KPI）

- **人工减负：** 电话咨询次数环比下降 30%。
- **留存效果：** 患者对复诊提醒的响应率（预约转化率）提升 20%。
- **无感维护：** 无需额外招聘运营，依靠医生录入即可实现全流程自动化。

### 给您的执行小贴士：

- **先从“最长流程”切入：** 建议优先上线“种植牙”或“正畸”流程，因为这两个项目周期长、节点多，也是对门诊利润贡献最大的，最值得用数字化手段去“锁住”客户。
- **关于小程序开发：** 您可以使用“微信开发者工具”，结合 **Vant Weapp** 组件库（这是一套开源的电商级组件），里面的“进度条”、“卡片”、“按钮”都是现成的，拖进去稍微调一下逻辑，不到三天就能做出一个能用的版本。
